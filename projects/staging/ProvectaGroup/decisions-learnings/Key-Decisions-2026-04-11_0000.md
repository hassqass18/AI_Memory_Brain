## Decisions

| Decision | Rationale |
|----------|-----------|
| Build AI Audit nurture sequence (6 emails, Days 0/3/7/12/18/25) using same brand system as RevOps sequence | Reuses existing brand infrastructure (logo, earth header, signature, CSS system). Consistent presentation across both sequences. |
| Host email images on pgco.world/assets/emails/ via Netlify API instead of base64 embedding | Base64 images are blocked/stripped by Gmail app, Apple Mail, and most mobile clients. Hosted URLs resolve on every client. Verified 200 responses from all 3 image URLs. |
| Zoho Campaigns image upload API not used — no public endpoint available with current scopes | API returns 1004 error on all attempted endpoints. Scopes (ZohoCampaigns.campaign.ALL, ZohoCampaigns.contact.ALL) do not include image library access. Alternative: user uploads via Zoho's UI Image Manager. |
| Switched email hero header from CSS background-image on `<td>` to plain `<img>` tag with baked-in tint | CSS background-image on `<td>` is stripped by Gmail app on Android/iOS — the single most common email rendering failure. Plain `<img>` renders everywhere. Blue tint composited into image file (55% navy overlay via PIL) since CSS rgba overlay is also stripped. |
| Removed overflow:hidden, box-shadow, border-radius from container table inline styles | overflow:hidden on tables suppresses image rendering in Samsung Mail and older Android clients. box-shadow and border-radius on tables cause clipping behavior. Kept in CSS-only (not inline) so capable clients still apply them. |
| Removed width="600" attribute from hero img — CSS-only sizing | Explicit width attribute forces 600px on a 375px screen. Some clients honor attribute over CSS width:100%, causing overflow and image-blocking. CSS-only max-width:600px;width:100% is the correct approach. |
| Added table-layout:fixed to container table | Ensures consistent column widths across email clients. Prevents layout shifts that can cause images to render outside their bounds. |
| Gold divider rebuilt as bgcolor table row (not CSS div) | CSS div inside table cell renders inconsistently across email clients. bgcolor attribute on a dedicated <td> renders in every client including Outlook. |
| Confirmed Zoho Campaigns merge tag syntax: $[FNAME]$, $[ORGANIZATION]$, $[DESIGNATION]$, $[LI:UNSUBSCRIBE]$ | Researched via Zoho official docs and community posts. All templates updated from incorrect {{}} format to correct $[]$ format. Custom fields use $[Field Label]$ matching exact Zoho field label. |
| Tinted earth header image cropped to 1200x260px (rendered 600x130px) | User requested smaller hero area. Compact strip maintains brand presence without dominating email layout. |
| Created standalone-previews/ folder with 11 individual HTML files | Allows user to open email files directly on mobile browser to test rendering before uploading to Zoho — bypasses iframe limitations of the reference viewer. |

## Learnings

- **Zoho Campaigns image API**: No public endpoint for image library upload. The `ZohoCampaigns.campaign.ALL` scope is insufficient. Workaround: upload via UI Image Manager in email editor (gets Zoho CDN URL) or use external hosted images.
- **Email mobile image rendering**: The 4 most common causes of images not showing on mobile — (1) CSS background-image on td, (2) width attribute conflicting with CSS width, (3) overflow:hidden on container tables, (4) missing table-layout:fixed.
- **Netlify deploy API**: To update a single file, must fetch full manifest from a ready deploy, add new file SHA, create new deploy, then upload required files. Cached files (matching SHA) don't need re-upload.
- **iframe srcdoc on mobile**: Mobile browsers (Safari, Chrome on iOS/Android) may not render large srcdoc iframes correctly. Standalone HTML files are needed for true mobile testing.
- **PIL image tinting**: PIL Image.blend() with alpha=0.55 for a solid color overlay produces reliable cross-client-compatible tinted photos without CSS dependency.
