## Learnings

### L-030: LinkedIn Creator Mode is no longer accessible via desktop in 2026
- **What happened:** Tried 5 different paths to reach Creator Mode (Resources dropdown, `/mypreferences/d/creator-mode` URL → 404, `/mypreferences/d/follower-settings` → 404, Settings > Visibility, Me menu). None worked.
- **Fix/Workaround:** Enable via LinkedIn mobile app → Profile → Resources → Creator Mode. The profile still shows follower count (1,347), suggesting Creator Mode features may already be partially active.
- **Why it matters:** Future LinkedIn automation work should not attempt to navigate to Creator Mode via desktop; direct users to mobile.

### L-031: Staging folders without core infrastructure files break AI Memory continuity
- **What happened:** 4 of 5 staging folders had rich content but no `Master-Context.md`, `Key-Decisions.md`, `Sessions.md`, `NextSteps.md`, or `agents/` — meaning any future AI session would have no structural anchor point.
- **Fix:** Always run the full staging.md Phase 2–4 (directory structure + core files + agents) when initializing a staging project. Don't let content accumulate before infrastructure is built.
- **Why it matters:** The AI Memory system's value depends on the infrastructure, not just the content. Rich docs in a structureless folder is still just a folder.
