## KD-011: Notion Content as Structured Text (Not Databases)

**Decision:** Add all project content to Notion as structured text/checkbox blocks rather than formal Notion databases.

**Context:** Notion database blocks (collection + collection_view) require complex CRDT-formatted schemas to create via the internal API. Text blocks, headings, and to_do checkboxes use a simple `set` command with title properties.

**Rationale:** Speed and reliability. Structured text with headings, bullets, and checkboxes is fully functional for Hassan's day-to-day use. Databases can be added later manually or via the MCP connector when available.

**Trade-off:** Less filtering/sorting capability vs. getting the content live fast.

**Status:** Confirmed

## KD-012: Notion Internal API via Browser Session Cookies

**Decision:** Use Notion's internal `/api/v3/saveTransactionsFanout` endpoint from the browser tab's session cookies rather than the public Notion API.

**Context:**
- Sandbox network proxy blocks outbound calls to `api.notion.com` (same restriction as GitHub)
- Notion public API also CORS-blocked when called from browser JavaScript on other domains
- Notion's own page already has auth session cookies (HttpOnly — not accessible to JS but sent automatically)

**Method:**
- Use `fetch('/api/v3/saveTransactionsFanout', { credentials: 'include', ... })` from within the notion.so tab
- This rides the authenticated browser session without needing to handle tokens

## Notion API Reference (Saved for Future Sessions)

### Block Creation Template
```json
{
  "pointer": { "table": "block", "id": "<newId>", "spaceId": "<spaceId>" },
  "path": [],
  "command": "set",
  "args": {
    "id": "<newId>",
    "type": "<type>",
    "properties": { "title": [["<text>"]] },
    "space_id": "<spaceId>",
    "created_time": <ms>,
    "created_by_table": "notion_user",
    "created_by_id": "<userId>",
    "last_edited_time": <ms>,
    "last_edited_by_id": "<userId>",
    "last_edited_by_table": "notion_user",
    "alive": true,
    "parent_id": "<parentId>",
    "parent_table": "block"
  }
}
```

### Block Types Used
- `header` — H1 heading
- `sub_header` — H2 heading
- `text` — paragraph
- `bulleted_list` — bullet point
- `to_do` — checkbox (properties: `checked: [["Yes"]] or [["No"]]`)
- `divider` — horizontal rule
- `callout` — callout block

### List Insertion
```json
{ "pointer": { "table": "block", "id": "<parentId>", "spaceId": "<spaceId>" }, "path": ["content"], "command": "listAfter", "args": { "id": "<newBlockId>", "after": "<previousBlockId>" } }
```

### Block Deletion
```json
[
  { "pointer": { "table": "block", "id": "<blockId>", "spaceId": "<spaceId>" }, "path": [], "command": "update", "args": { "alive": false } },
  { "pointer": { "id": "<parentId>", "table": "block", "spaceId": "<spaceId>" }, "path": ["content"], "command": "listRemove", "args": { "id": "<blockId>" } }
]
```
