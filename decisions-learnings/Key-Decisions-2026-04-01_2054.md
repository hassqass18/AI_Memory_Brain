## Decisions

### D-015: "American Dream Apartments" adopted as primary development name in copy
- **Context:** Hassan provided an official copy block using "American Dream Apartments" as the primary name
- **Decision:** Use "American Dream Apartments" consistently as the development name throughout the website. "Sierra Homes American Dream" used where parent company attribution is needed (footer, legal line)
- **Rationale:** Cleaner, more brandable; separates development identity from the parent company Sierra Homes. Aligns with the American lifestyle positioning.

### D-016: Pricing removed from all public-facing copy — enquire/request-only model
- **Context:** Website previously showed KES unit prices (1BR: KES 6M, 2BR: KES 13M, 3BR: KES 18M) and projected 2029 values
- **Decision:** Remove all exact KES figures. Investment table now shows projected capital gain % and rental yield % only. All CTAs read "Request Pricing"
- **Rationale:** Preserves negotiation flexibility, avoids early market anchoring, allows sales team to qualify and engage leads before disclosing price. Standard practice for premium off-plan developments

### D-017: EcoTecture removed from all public website copy
- **Context:** Architect attribution appeared in showcase section header, services section, intro copy, and footer
- **Decision:** Remove all EcoTecture mentions from the public-facing website. Design copy reframed around outcomes (sunlight, ocean views, U.S. standards) not the firm
- **Rationale:** Architect attribution is not strategically important for the buyer or investor journey. Focus belongs on the development brand and the lifestyle/investment proposition

### D-018: Tower names removed from primary floor plan navigation
- **Context:** Floor plan tabs were labelled "3BR · Lincoln" and "3BR · Washington"; tower name appeared as primary identifier throughout floor plan section
- **Decision:** Tabs now read "3 Bedroom — Ocean View" and "3 Bedroom — Garden View". Tower names removed from diagram overlays. Tower badges replaced with spec + orientation labels. Tower mentioned only within descriptive copy as secondary context
- **Rationale:** Floor plans are the product being sold, not the towers. Orientation is the meaningful differentiator for buyers. Tower naming is internal structural context

### D-019: Header changed from fixed to relative positioning
- **Context:** Navbar was `position: fixed`, overlaying hero content as a persistent floating element
- **Decision:** Changed to `position: relative` with always-visible navy background. Header now scrolls with the page
- **Rationale:** Hassan's direct instruction — header should move with the rest of the site, not stay pinned at the top

### D-020: Two contact numbers published on site (USA + Kenya offices)

- **Decision:** USA Office: +1 407-979-4473 | Kenya Office: +254 733-555501 | Email: sales@sierrahomesusa.com

- **Observation:** Hassan referenced a prior session where naming conventions and copy were changed. No session summary existed for this. The changes were not in the file either — suggesting the session was conversational only or changes were not saved

- **Observation:** Used `replace_all: true` to replace "Register Interest" with "Download Brochure" across all occurrences (nav + mobile drawer) in a single operation
