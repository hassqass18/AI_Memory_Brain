### D-011 — Leaflet.js + CartoDB Light Tiles for Neighbourhood Map
**Decision:** Use Leaflet.js (v1.9.4 via cdnjs) with CartoDB Light basemap for the interactive Nyali neighbourhood map.
**Rationale:** (1) No API key required — both Leaflet and CartoDB Light tiles are open-source/free-to-use in a single HTML file. (2) CartoDB Light tile style is minimal and professional — white background, subtle labels, no visual noise — fitting the premium brand aesthetic. (3) Custom div icons via `L.divIcon` allow full CSS control of marker appearance (emoji + background color + border-radius).
**Implementation:** 16 pins across 7 colour-coded categories. Dashed 2-mile radius circle (`L.circle` with `dashArray`). Custom popup HTML with category label, place name, description, and distance. `scrollWheelZoom: false` to prevent accidental zoom during page scroll.

## Learnings

### L-006 — Leaflet Map Requires `scrollWheelZoom: false` on Landing Pages
**Learning:** When embedding Leaflet maps in a full-page scroll layout, the default `scrollWheelZoom: true` causes the map to zoom when a user scrolls past it — disorienting UX. Always set `scrollWheelZoom: false` for embedded maps on scrolling pages.
**Fix:** `L.map('id', { scrollWheelZoom: false })` in the Leaflet constructor.

### L-007 — Leaflet `L.divIcon` className Must Be Empty String for Custom HTML Icons
**Learning:** If `L.divIcon` is called with a non-empty `className`, Leaflet adds default icon CSS that conflicts with custom marker styling. Setting `className: ''` gives full CSS control to the custom HTML element.
**Fix:** Always use `className: ''` when using custom HTML markers via `L.divIcon`.

### L-008 — SVG `text` Elements Need Explicit Font-Family for Brand Fonts
**Learning:** SVG `<text>` elements do not inherit CSS custom properties or Google Fonts loaded in the parent HTML unless the font-family is explicitly declared inline on the SVG element. Without this, SVG text falls back to the browser's default serif.
**Fix:** Always set `font-family="Outfit,sans-serif"` (or the desired font) directly on SVG `<text>` elements.

### L-009 — Tab Panel Fade-Up Animations Must Be Re-Triggered on Tab Change
**Learning:** IntersectionObserver marks elements as `visible` on first view and then `unobserve()`s them. When a tab panel is hidden (`display: none`) and then shown (`display: grid`), the fade-up elements are already marked visible but may not have played the animation. On tab switch, manually remove and re-add the `visible` class with a short timeout.
**Fix:** In tab click handler: `el.classList.remove('visible'); setTimeout(() => el.classList.add('visible'), 50);`
