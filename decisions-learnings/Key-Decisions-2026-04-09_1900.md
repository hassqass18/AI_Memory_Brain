## Learnings

**L-032: Netlify 404 if no index.html at zip root**
- Uploading a zip with only named HTML files (e.g., `provecta-group.html`) to Netlify returns "Page not found" because Netlify serves `index.html` by default with no redirect rules
- **Fix:** Always rename the primary HTML file to `index.html` before zipping for Netlify. Add a `_redirects` file if you need custom routing (e.g., `/* /index.html 200` for SPAs).
- **Applies to:** Netlify drag-and-drop zip deploys, GitHub Pages, any static host without explicit routing config.

**L-033: Chrome in Chrome MCP navigated to live provectagroup.com instead of local file**
- When asked "can you see the Provecta Group website?" the browser tool navigated to the live domain
- Hassan clarified he meant the local HTML file in the AI Memory Brain folder
- **Fix:** When a user says "the [project] website" within a project context, default to checking the local staging file first. Use "live site" or a URL for the hosted version.
