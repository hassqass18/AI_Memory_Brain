## Learnings

### L-002 — Notion Reverse Relations Require Explicit Update
**Issue:** When creating Notion tasks via API with the `Projects` relation field set, the project's `Tasks` reverse relation field does NOT auto-populate.
**Fix:** After creating tasks, make a separate `notion-update-page` call on the project page, passing a JSON-encoded array string of all task URLs in the `Tasks` property. Format: `"[\"url1\",\"url2\",...]"` — NOT a native array (tool only accepts string/number/null values).
**Applies to:** Any future Notion project/task linking via API.

### L-003 — npm Global Install Blocked in Cowork Sandbox
**Issue:** `npm install -g docx` fails with permission error in Cowork sandbox environment.
**Fix:** Create a local working directory, run `npm init -y` + `npm install docx` locally, run script from that directory.
**Applies to:** Any npm package usage in Cowork sessions.

### L-004 — Notion replace_content Must Preserve Child Pages
**Issue:** `notion-update-page` with `replace_content` command fails if the page has child pages and the `new_str` does not include them via `<page url="...">` tags.
**Fix:** Use `update_content` (search-and-replace) to add content above existing child page references, OR include `<page url="...">` tags in `new_str` to preserve child pages.
**Applies to:** Any Notion page content update where child pages exist.

### L-005 — validate.py Path Resolution in docx Skill
**Issue:** Default relative path `/skills/docx/../../../scripts/office/validate.py` returns file-not-found.
**Fix:** Use `find` to locate the script: actual path is `/sessions/.../mnt/.claude/skills/docx/scripts/office/validate.py`.
**Applies to:** Any session using the docx skill for document generation.
