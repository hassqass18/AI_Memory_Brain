## Learnings

| ID | Learning | Detail |
|----|----------|--------|
| L-011 | V-Ray vs Lumion quality gap is significant | Lumion (popular in EA market) uses rasterisation with screen-space effects — produces soft, slightly plastic look. V-Ray/Corona use true ray tracing. Must specify engine in brief, not just "high-quality renders." |
| L-012 | Lighting time-of-day is a brief item, not a creative choice left to studio | Golden hour renders (18:30 EAT Nyali) must be explicitly spec'd with correct sun angle for Mombasa latitude (~4°S). Leaving this to the studio risks noon flat lighting by default. |
| L-013 | Indian Ocean visible horizon is a unique SierraHomes visual asset | The project's Nyali location means the ocean should appear in aerial and high-angle renders. This must be explicitly called out in the brief — it is a differentiator no Nairobi project can claim. |
| L-014 | Never preview HTML/website files in the browser — render within Claude instead | Attempting to screenshot localhost URLs via Claude in Chrome always fails ("Frame with ID 0 showing error page") because the sandboxed browser blocks localhost connections. It also wastes significant tokens navigating, waiting, retrying, and spinning up HTTP servers. **Standard practice going forward: always serve HTML output as a `computer://` file link so Hassan opens it directly in his browser. No HTTP server, no screenshot loop.** Applies to: website redesigns, HTML prototypes, email templates, landing pages, any .html deliverable. |
