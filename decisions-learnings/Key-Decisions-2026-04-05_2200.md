## Learnings

### L-019: LinkedIn direct WebFetch returns HTTP 999 (blocked)
**What happened:** WebFetch call to linkedin.com/in/alexander-orta returned status 999 — LinkedIn's anti-scraping block.
**Fix:** Use Claude in Chrome browser automation to navigate to the profile directly. However, see L-020.

### L-020: LinkedIn Chrome automation returns only headline/activity — full profile sections do not load
**What happened:** Even after navigating via Chrome, the Experience and Education sections were not present in the accessibility tree. Only the hero/headline and activity feed were accessible.
**Root cause:** LinkedIn uses lazy-loading and likely serves limited data to non-connections or requires additional scrolling triggers.
**Fix:** Supplement LinkedIn with about.me, web search (`"[name]" site:about.me`), and direct request to the subject for a copy of their full profile or CV.

### L-021: about.me profiles can contain rich career data not visible on LinkedIn
**What happened:** `about.me/alexander_orta` returned Alex's career statement, Peace Corps role, Liquidstar role, and UF undergraduate — none of which were visible from LinkedIn alone.
**Fix:** Always search `[name] site:about.me` as a standard step in partner/contact research.

### L-022: Notion create-pages for database entries requires data_source_id, not database_id
**What happened:** Initial attempt to add records to MSIFS Action Log database failed because database_id was used instead of data_source_id.
**Fix:** Run `notion-fetch` on the database URL first. Look for `<data-source url="collection://[uuid]">` in the response. Pass that UUID as `data_source_id` in the `parent` object.

### L-023: Notion update-page replace_content requires `new_str` parameter, not `content`
**What happened:** First call to `notion-update-page` with `command: replace_content` and `content:` parameter returned validation error.
**Fix:** The correct parameter for `replace_content` command is `new_str`, not `content`. Always check tool error message — it states the required parameter name clearly.
