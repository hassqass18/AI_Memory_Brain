## Learnings

### L-015: LinkedIn Blocks Web Scraping (Status 999)
- **Issue:** WebFetch to LinkedIn profile returned status 999 (bot detection)
- **Workaround:** Used uploaded .docx profile content and Python docx library for text extraction
- **Future:** Always extract LinkedIn data from downloaded/uploaded profile documents, not live scraping

### L-016: Chrome JS Extraction Blocked by Image URLs with Query Params
- **Issue:** Third JavaScript extraction attempt on Gilber theme returned `[BLOCKED: Cookie/query string data]` when extracting image src URLs
- **Workaround:** Avoid extracting image src URLs in JS scripts; focus on computed styles, text content, and class names
- **Future:** When extracting design data via Chrome JS, explicitly exclude `src`, `href`, and URL-containing attributes

### L-017: Binary .docx Files Cannot Be Read with Read Tool
- **Issue:** Read tool returned garbled binary content for .docx files
- **Workaround:** Used `python3 -c "from docx import Document; ..."` via Bash to extract text programmatically
- **Future:** Always use Python docx library for Word document text extraction in Cowork

### L-018: Cowork Conversation Images Not Saved as Files on Disk
- **Issue:** Images uploaded inline in conversation are visible to the model as base64 but not accessible as files in `/sessions/` or `/tmp/`
- **Workaround:** Created placeholder images with labels using Python PIL; instructed user to replace files manually
- **Future:** For image-dependent websites, create named placeholders and provide clear renaming instructions to user
